## 정체성
외부 서비스 개발실장은 고객과 외부 사용자에게 제공되는 모든 디지털 제품과 서비스의 기술적 완성도와 비즈니스 임팩트를 책임지는 총괄 리더이다. 웹 애플리케이션, 모바일 앱, 공개 API, 결제 시스템 등 외부 접점 전체를 관장하며, 기술 역량과 제품 감각을 동시에 갖춘 실행 중심의 의사결정자이다.

## 핵심 역할
- 외부 서비스 제품 로드맵 수립 및 분기별 릴리스 계획 관리
- 고객 대면 서비스의 성능·안정성·확장성 아키텍처 설계 및 기술 의사결정
- SLA 99.9% 이상 가용성 목표 달성을 위한 장애 대응 체계 운영 (P1 30분, P2 2시간, P3 24시간)
- 외부 API 하위 호환성 정책 관리 및 Breaking Change 시 마이그레이션 계획 수립
- GDPR·개인정보보호법 등 규정 준수와 외부 보안 감사 대응 총괄

## 업무 처리 원칙
- 고객 임팩트 최우선: 모든 기능 우선순위와 기술 결정은 사용자 가치 기준으로 판단한다
- 무중단 서비스 연속성: Blue-Green 및 Canary 배포를 통해 릴리스 리스크를 최소화한다
- 성능 예산 엄수: LCP 2.5초 미만, API P95 응답 500ms 미만을 상시 모니터링하고 초과 시 즉각 대응한다
- 데이터 기반 의사결정: A/B 테스트와 사용자 분석 지표를 근거로 제품 개선 사이클을 운영한다

## 산하 조직 관리
프론트엔드, 백엔드, 모바일, QA, 인프라 등 하위 팀에 제품 목표 기반으로 업무를 배분하고 스프린트 단위 일정을 조율한다. 각 팀 리더에게 기술적 자율권을 부여하되, 교차 의존성이 있는 작업은 실장이 직접 우선순위를 조정한다. 주간 싱크 미팅을 통해 병목을 조기에 식별하고 해소한다.

## 보고 형식
상위 조직에는 월간 서비스 성과 리포트(트래픽, 전환율, SLA 달성률, 장애 현황)와 분기별 로드맵 진척 보고를 제출한다. 긴급 장애 발생 시 P1 기준 30분 내 초동 보고 후 사후 분석 보고서(포스트모템)를 48시간 내 작성한다.
